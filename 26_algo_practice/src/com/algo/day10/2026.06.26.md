## 문제 1. K번째 수 (백준 1300)

### 문제 요약

- N×N 곱셈표 A에서 A[i][j] = i×j.
- 이 값들을 1차원 배열 B에 모두 담아 오름차순 정렬했을 때 B[k]를 구한다.
- 인덱스는 1부터 시작.
- 제한: N ≤ 10⁵, k ≤ min(10⁹, N²). → N² 최대 10¹⁰ (long 필요).

---

### 내 풀이 분석

- **풀이 결과**: 실패 — 입력만 읽고 알고리즘 미구현.

**알고리즘**

- 정렬은 N² = 10¹⁰ 규모라 불가능함을 인지하고, 힙 자료구조를 떠올렸으나 적용 방법을 찾지 못함.

**시간복잡도**

- 미구현.

**✅ 잘 짚은 것**

- N² 범위가 long(10¹⁰)임을 정확히 파악.
- 단순 정렬은 시간초과라는 점을 인지.

**❌ 문제점 / 개선점**

- 힙을 떠올린 방향이 함정. 힙으로 K번째를 뽑아도 원소를 N²개 다뤄야 해 여전히 O(N² log) → 불가능.
- 이 문제의 핵심은 "자료구조"가 아니라 **답을 이분 탐색**하는 발상이다.

---

### 모범 답안: 정답 이분 탐색 (Parametric Search)

**알고리즘**

1. 답 x의 범위는 [1, N²]. 이 x를 직접 이분 탐색한다.
2. 판정 함수 cnt(x) = "곱셈표에서 값이 x 이하인 원소 개수".
   - i번째 행은 i, 2i, 3i, … 이므로 x 이하 개수는 `min(x/i, n)`.
   - cnt(x) = Σ_{i=1..n} min(x/i, n).
3. cnt(x)는 x에 대해 단조 증가. `cnt(x) >= k`를 만족하는 **가장 작은 x**가 답.
   - 그 x는 반드시 어떤 i×j 값이 되므로 B[k]와 일치한다.

**예시 추적** (N=3, k=7)

```
곱셈표:        정렬된 B:
1 2 3          1 2 2 3 3 4 6 6 9
2 4 6          B[7] = 6
3 6 9

cnt(6) = min(6/1,3)+min(6/2,3)+min(6/3,3) = 3+3+2 = 8 >= 7
cnt(5) = min(5/1,3)+min(5/2,3)+min(5/3,3) = 3+2+1 = 6 < 7
→ 답 = 6
```

**시간복잡도**

| 단계 | 복잡도 |
|------|--------|
| 이분 탐색 반복 | O(log N²) = O(log N) |
| 매 반복 개수 세기 | O(N) |
| 합계 | O(N log N) |

**공간복잡도**: O(1)

---

### 핵심 정리

- "B[k]를 직접 구하라"처럼 **k번째 값**을 묻는데 후보 공간이 거대하면, 값을 만들지 말고 **답을 이분 탐색** + **"x 이하 개수 세기"** 판정 패턴을 떠올린다 (Parametric Search).
- cnt(x) >= k 의 최소 x를 찾으면, 그 x가 자연스럽게 k번째 원소가 된다 (lower_bound 논리).
- 곱셈표의 행은 등차수열이라 한 행의 "x 이하 개수"가 `min(x/i, n)`으로 O(1)에 계산된다는 점이 이 문제의 열쇠.

## 문제 2. 팰린드롬 분할 (백준 1509)

### 문제 요약

- 문자열을 연속 구간들로 나누되, 각 조각이 모두 팰린드롬이어야 한다.
- 분할 조각 개수의 **최솟값**을 출력한다.
- 입력: 알파벳 대문자, 최대 길이 2,500.

---

### 내 풀이 분석

- **풀이 시간**: 1m 29s 고민
- **풀이 결과**: 실패 — 방향은 잡았으나 코드 미구현.

**알고리즘**

- 최대 분할 수 = 문자열 길이라는 상한을 인지.
- `DP[i]` = 앞 i개를 분할했을 때 최소 조각 수 라는 1차원 DP를 떠올림.
- 다만 "구간이 팰린드롬인지 판단하는 부분"에서 막혀 구현으로 이어가지 못함.

**시간복잡도**

- 미구현.

**✅ 잘 짚은 것**

- DP 문제임을 정확히 직감 (연속 구간 분할 + 최소화).
- `DP[i] = 앞 i개 최소 분할`이라는 상태 정의가 정답과 동일.

**❌ 문제점 / 개선점**

- 막힌 지점("팰린드롬 판단")은 DP 전이와 **분리**해서 생각하면 풀린다.
- 모든 구간의 팰린드롬 여부를 **별도 2차원 표**로 미리 채워두면, DP 전이 때 O(1)로 조회 가능.

---

### 모범 답안: 팰린드롬 판정표 + 1차원 DP

**알고리즘**

1. `pal[i][j]` = 구간 [i..j]가 팰린드롬인가. 구간 길이를 1→N으로 키우며 채운다.
   - `s[i]==s[j]` 이고 (길이 ≤ 2 또는 `pal[i+1][j-1]`) 이면 팰린드롬.
2. `dp[i]` = 앞 i글자의 최소 팰린드롬 분할 수, `dp[0]=0`.
   - `dp[i] = min_{j<=i, pal[j..i]} ( dp[j-1] + 1 )`.
3. 답은 `dp[N]`.

**예시 추적** (AAAA)

```
pal[1..4] = true (전체가 팰린드롬)
dp[4] = dp[0] + 1 = 1   → 출력 1
```

**시간복잡도**

| 단계 | 복잡도 |
|------|--------|
| 팰린드롬 판정표 채우기 | O(N²) |
| DP 전이 | O(N²) |
| 합계 | O(N²) |

**공간복잡도**: O(N²) — 판정표.

---

### 핵심 정리

- "연속 구간으로 분할 + 조각 수/비용 최소화" → 거의 **1차원 DP** (`dp[i] = 앞 i개 최적값`).
- DP 전이에 "구간이 조건을 만족하는가" 판단이 필요하면, 그 판정을 **전처리 표(2차원)**로 분리하라. 본 문제의 팰린드롬 판정표가 전형적인 예.
- 팰린드롬 판정표는 **길이 오름차순(또는 i 내림차순)**으로 채워야 안쪽(`pal[i+1][j-1]`)이 먼저 계산된다.
