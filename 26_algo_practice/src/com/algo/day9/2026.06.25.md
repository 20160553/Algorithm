## 문제 1. 냉장 배송 상자 포장

### 문제 요약

- 컨베이어 벨트 순서대로 놓인 N개의 식품을 **연속한 구간 단위**로 잘라 상자에 담는다 (순서 변경/건너뛰기 불가).
- 한 상자에 담는 식품의 총무게는 W 이하.
- 상자 하나의 비용 = 기본 배송비 B + (그 상자에 담긴 식품 중 **최대 냉장 비용**).
- 모든 식품을 포장하는 **최소 총비용**을 구한다.
- 제한: 1 ≤ N ≤ 2,000 / 1 ≤ W ≤ 10,000 / 정답은 32비트를 넘을 수 있음 → **long 사용**.

> ⚠️ 예시 오류: 문제의 예제 출력 `250`은 잘못되었고, 정확한 최소 비용은 **330**이다. (문제 주석에도 동일하게 명시되어 있음)

---

### 내 풀이 분석

- **풀이 결과**: 실패 (입력 파싱까지만 작성, DP 미구현)

**알고리즘**

`dp[i][j]`를 "i~j 구간 내 최소값"으로 정의하려 시도했으나, 2차원 구간 DP로 접근하면서 점화식을 세우지 못하고 멈췄다. "몇 번 잘라야 할지 모른다"는 점에서 분할 DP라는 감은 잡았다.

**✅ 잘 짚은 것**

- 이 문제가 **DP(구간 분할)** 라는 큰 방향은 정확히 인지했다.
- "연속 구간만 한 상자에 담을 수 있다"는 핵심 제약을 이해했다.

**❌ 문제점 / 개선점**

- `dp[i][j]`(구간 최소)라는 2차원 상태가 과한 설계였다. 이 문제는 "어디서 자르느냐"만 결정하면 되므로 **끝점 하나만 상태로 두는 1차원 DP**로 충분하다.
- 점화식의 핵심은 "마지막 상자가 어디서 시작하는가(j)"를 모든 후보에 대해 시도하는 것인데, 이 분해를 떠올리지 못했다.
- 정답이 32비트를 넘는다는 조건을 코드에 반영하지 못했다 (long 필요).

---

### 모범 답안: 1차원 구간 분할 DP

**알고리즘**

1. `dp[i]` = 앞에서부터 i개의 식품을 모두 포장하는 최소 비용. `dp[0] = 0`.
2. i를 1부터 N까지 올리면서, **마지막 상자가 (j+1 .. i)를 담는다**고 가정한다.
3. j를 `i`부터 거꾸로 내리며 무게 합과 구간 최대 냉장비용을 누적한다.
   - 무게 합이 W를 넘으면 더 내려갈 필요가 없으므로 `break`.
   - `dp[i] = min(dp[i], dp[j-1] + B + maxCost(j..i))`.
4. 답은 `dp[N]`.

j를 거꾸로 내리며 누적하므로 구간 최댓값을 매번 다시 계산하지 않고 O(1)로 갱신할 수 있는 것이 포인트다.

**예시 추적** (N=5, W=10, B=100 / weight,cost = (4,30)(2,80)(5,20)(3,50)(2,40))

```
dp[0]=0
dp[1]=130            (식품1 단독: 100+30)
dp[2]=180            (식품1,2 한 상자: 100+max(30,80))
dp[3]=300            (dp[2]+100+20)
dp[4]=310            (dp[1]+100+max(80,20,50), 식품2~4 한 상자)
dp[5]=330            (dp[2]+100+max(20,50,40), 식품3~5 한 상자)  → 정답 330
```

**시간복잡도**

| 단계 | 복잡도 |
|------|--------|
| 바깥 i 루프 | O(N) |
| 안쪽 j 루프(무게 초과 시 break) | O(N) |
| 합계 | O(N²) ≈ 4×10⁶ |

**공간복잡도**: O(N)

---

### 핵심 정리

- "**연속 구간을 여러 조각으로 나눠 각 조각의 비용 합을 최소화**" → 끝점 하나를 상태로 두는 1차원 분할 DP의 전형적 패턴: `dp[i] = min_j ( dp[j-1] + cost(j..i) )`.
- 구간 DP라고 무조건 2차원 `dp[i][j]`로 갈 필요는 없다. "잘리는 지점"만 중요하면 1차원으로 내려간다.
- 안쪽 루프를 **끝점에서 거꾸로** 돌리면 구간의 합·최댓값을 누적으로 O(1) 유지 → 전체 O(N²).
- 무게/길이 같은 단조 증가 제약은 초과하는 순간 `break`로 가지치기.
- "정답이 32비트를 넘을 수 있음" 같은 문구는 곧 **long 선언** 신호다.

## 문제 2. 외판원 순회 (BOJ 2098)

### 문제 요약

- 1~N번 도시가 있고 `W[i][j]`는 i→j 이동 비용. `W[i][j] = 0`이면 **길이 없음**(이동 불가).
- 한 도시에서 출발해 **모든 도시를 정확히 한 번씩** 방문하고 **출발 도시로 복귀**하는 순회(Hamiltonian cycle)의 최소 비용을 구한다.
- 제한: 2 ≤ N ≤ 16 → 비트마스크 DP 신호. 비용 ≤ 1,000,000.
- 항상 순회 가능한 입력만 주어진다.

---

### 내 풀이 분석

- **풀이 결과**: 부분 성공 — 컴파일/실행되고 예제(35)는 맞지만, **논리적으로 틀린 답을 내는 입력이 존재**한다.

**알고리즘**

비트마스크 TSP DP. `costs[mask][last]` = mask 집합을 방문하고 last에 있는 최소 비용. 모든 도시 i를 시작점 후보로 두고(`costs[1<<i][i]=0`), 전이 후 마지막에 `costs[FULL-1][last] + map[last][start]`로 닫았다.

**시간복잡도**

- O(2^N · N²) — N=16에서 적정. 이 부분은 맞다.

**✅ 잘 짚은 것**

- N ≤ 16을 보고 **비트마스크 DP**를 정확히 골랐다.
- `dp[mask][last]` 상태 정의와 전이(다음 도시 추가) 구조 자체는 표준적이고 올바르다.
- `INF = Integer.MAX_VALUE / 2`로 오버플로를 피한 점, 방문 비트 체크 등 디테일이 깔끔하다.

**❌ 문제점 / 개선점**

- **(치명적) 복귀 간선이 실제 출발지로 가지 않는다.** 시작점을 고정하지 않고 모든 도시를 시작점으로 두면 `costs[FULL-1][last]`를 만든 경로의 *실제 출발지*를 알 수 없다. 그런데 마지막에 `map[last][start]`의 `start`를 임의 도시로 최소화하므로, 출발지로 돌아오지 않는 **비-순환 경로(경로 + 엉뚱한 꼬리 간선)** 까지 후보가 되어 답을 **과소평가**한다.
  - 실측: `0`번으로 들어오는 간선만 비싸게 만든 입력에서 정답은 **1003**인데 사용자 코드는 **4**를 출력.
  - 해결: 순회는 시작점을 어디로 잡아도 같으므로 **시작점을 0으로 고정**하고, 닫는 간선은 항상 `map[last][0]`으로 보낸다.
- **`W[i][j] == 0`(길 없음)을 비용 0으로 더한다.** 막힌 길을 공짜 이동으로 취급하는 버그. 전이/복귀 모두에서 `0`이면 `continue`로 건너뛰어야 한다. (예제엔 막힌 길이 없어 드러나지 않음)
- `Arrays.fill(costs[0], 0)`은 사용되지 않는 불필요한 초기화(mask 0은 어떤 last 비트도 없어 전이에 쓰이지 않음).

---

### 모범 답안: 비트마스크 TSP DP (시작점 0 고정)

**알고리즘**

1. `dp[mask][last]` = 0번에서 출발해 `mask`를 모두 방문하고 `last`에 있는 최소 비용. `dp[1][0] = 0`.
2. 0번을 포함하지 않는 mask는 건너뛴다(`(mask & 1) == 0`).
3. 전이: `last`에서 아직 안 간 `next`로, 단 `map[last][next] != 0`(길 존재)일 때만
   `dp[mask|1<<next][next] = min(..., dp[mask][last] + map[last][next])`.
4. 답: 모든 도시를 방문한 상태에서 0으로 복귀 — `min over last ( dp[FULL-1][last] + map[last][0] )`, 단 `map[last][0] != 0`.

**시간복잡도**

| 단계 | 복잡도 |
|------|--------|
| mask × last × next 전이 | O(2^N · N²) |
| 합계 | O(2^N · N²) ≈ 1.7×10⁷ (N=16) |

**공간복잡도**: O(2^N · N)

---

### 핵심 정리

- N ≤ 16~20이면 거의 항상 **비트마스크 DP** 신호.
- **TSP는 시작점을 한 곳(보통 0번)으로 고정**한다. 순회는 회전 대칭이라 시작점을 고정해도 최솟값은 같고, 무엇보다 "복귀 간선이 반드시 출발지로 가야 한다"는 제약을 코드로 강제할 수 있다. 시작점을 고정하지 않으면 닫는 간선이 엉뚱한 도시로 새서 답이 과소평가된다.
- "길이 없음 = 0"처럼 **0이 '없음'을 뜻하는 입력**은 0을 비용으로 더하지 말고 반드시 `continue`로 건너뛴다. 예제에 막힌 길이 없으면 버그가 안 드러나니 주의.
- 예제를 통과해도 정답이 아닐 수 있다 → **반례(특히 비대칭/막힌 길) 직접 구성 + 브루트포스 대조**로 검증하는 습관.
