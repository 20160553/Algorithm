## 문제 1. 구독 서비스 정산하기

### 문제 요약

- 길이 `N`의 정수 배열 `A`가 주어진다 (양수=결제, 음수=환불, 0=무변동).
- 연속 구간 중 **합이 정확히 X**이고 **길이가 L 이상**인 구간의 개수를 센다.
- 정답은 64비트 정수 범위.
- 제한: `1 ≤ N ≤ 500,000`, `-10^9 ≤ A[i] ≤ 10^9`, `-10^14 ≤ X ≤ 10^14`, `1 ≤ L ≤ N`.

---

### 내 풀이 분석

- **풀이 결과**: 실패 — 입력 파싱까지만 작성하고 본 알고리즘은 미구현. `main` 끝에서 멈춤.

**알고리즘**

작성된 풀이 로직은 없고, 주석으로 접근 방향을 탐색한 흔적만 있다.

> "완탐 N² = 5×10¹⁰ 절대 아님 / 누적합 + DP인 것 같긴 한데... 분할 정복? 트리 누적합?"
> "누적합으로 합산을 줄여도 left, right 연산만 해도 10¹⁰을 넘는데 시간 안에 되나?"

**✅ 잘 짚은 것**

- **O(N²) 완전탐색이 불가능**함을 제한(N=5×10⁵)으로부터 정확히 판단했다.
- **누적합(prefix sum)** 을 핵심 도구로 떠올린 것은 정답 방향이다.

**❌ 문제점 / 개선점**

- 마지막 주석의 오해가 결정적이다. "left, right 연산만 해도 10¹⁰"이라는 건 여전히 **모든 (i, j) 쌍을 도는 O(N²)** 발상에 갇혀 있다는 뜻이다.
- 핵심 전환: 구간 합 `P[j] - P[i] = X` 를 **"각 j마다 i를 다시 도는 문제"가 아니라 "P[j] - X 값이 과거에 몇 번 나왔는가"를 묻는 문제**로 바꾸면, 쌍을 도는 대신 **해시맵 조회 1회**로 끝난다 → O(N).
- 길이 제약 `j - i ≥ L` 을 어떻게 끼울지 고민이 멈춤의 원인으로 보이는데, "i = j−L 시점에 비로소 P[i]를 맵에 등록"하는 **지연 등록**으로 추가 루프 없이 처리된다.

---

### 모범 답안: 누적합 + 해시맵 (지연 등록)

**알고리즘**

1. 누적합 `P[k] = A[0] + ... + A[k-1]` 를 계산한다 (`P[0] = 0`).
2. 구간 `(i+1 .. j)` 의 합은 `P[j] - P[i]`. 합 = X 조건은 `P[i] = P[j] - X`.
3. 길이 조건 `j - i ≥ L` ⇔ `i ≤ j - L`. 즉 우측 끝 j를 볼 때 **유효한 i는 j−L 이하**.
4. j를 `L`부터 `N`까지 진행하며:
   - 새로 유효해진 인덱스 `i = j - L` 의 값 `P[j-L]` 을 해시맵에 등록(빈도 +1).
   - 맵에서 `P[j] - X` 의 빈도를 답에 누적.

**예시 추적** (`N=8, X=3, L=3`, `A = 1 2 -1 2 1 -2 3 0`)

```
P = [0, 1, 3, 2, 4, 5, 3, 6, 6]   (P[0..8])

j=3: 등록 P[0]=0 ; 조회 P[3]-X = -1 → 0건   (누적 0)
j=4: 등록 P[1]=1 ; 조회 P[4]-X =  1 → 1건   (누적 1)  구간 A[1..3]=2,-1,2
j=5: 등록 P[2]=3 ; 조회 P[5]-X =  2 → 0건   (누적 1)
j=6: 등록 P[3]=2 ; 조회 P[6]-X =  0 → 1건   (누적 2)  구간 A[0..5]
j=7: 등록 P[4]=4 ; 조회 P[7]-X =  3 → 1건   (누적 3)  구간 A[2..6]
j=8: 등록 P[5]=5 ; 조회 P[8]-X =  3 → 1건   (누적 4)  구간 A[2..7]
→ 정답 4
```

**시간복잡도**

| 단계 | 복잡도 |
|------|--------|
| 누적합 계산 | O(N) |
| j 순회 + 맵 조회/등록 | O(N) (평균 O(1) × N) |
| 합계 | **O(N)** |

**공간복잡도**: O(N) (누적합 값의 분포를 담는 해시맵)

---

### 핵심 정리

- **"연속 구간 합 = K 인 구간의 개수"는 누적합 + 해시맵의 정석 패턴이다.** `P[j] - P[i] = K` 를 `P[i] = P[j] - K` 로 뒤집어, 쌍을 도는 대신 "과거 값의 빈도"를 묻는 O(N) 문제로 바꾼다.
- **음수가 섞여 있으면 투 포인터/슬라이딩 윈도우는 단조성이 깨져 쓸 수 없다** → 해시맵 누적합이 정답.
- **"최소 길이 L 이상" 같은 거리 제약은 지연 등록으로 해결한다**: 우측 끝이 j일 때 등록 대상은 항상 `i = j - L` 하나뿐이라 추가 루프가 필요 없다.
- 누적합과 X 모두 64비트 범위를 넘나들므로 `long` 사용은 필수.

## 문제 2. 관광지 도보 투어 (열린 경로형 TSP)

### 문제 요약

- `n`개의 지점을 **정확히 한 번씩** 방문하는 최소 비용 경로를 구한다.
- day3(복귀형 TSP)과 달리 **출발지·도착지가 자유**이고, **출발지로 복귀하지 않는다**(열린 해밀턴 경로).
- `dist[i][j]`는 i→j 직접 이동 비용, 길 없음은 `-1`. 비대칭(`dist[i][j] ≠ dist[j][i]`) 가능.
- 모든 지점을 잇는 경로가 없으면 `-1`.
- 제한: `2 ≤ n ≤ 16`, `1 ≤ dist[i][j] ≤ 1,000,000` 또는 `-1`.

---

### 내 풀이 분석

- **풀이 결과**: 성공 — 3개 케이스 전부 PASS, 정답.

**알고리즘**

Held-Karp 비트마스크 DP. `dp[mask][last]` = 집합 mask를 방문하고 현재 last에 있을 때 최소 비용.
시작점을 자유롭게 하기 위해 **시작점 `start`를 0..n-1로 바꿔가며 매번 dp를 초기화 후 전체 DP를 재실행**하고, 각 실행의 `dp[FULL-1][*]` 최솟값을 누적했다.

**시간복잡도**

- `for start` 외부 루프(n) × Held-Karp(O(2^n · n^2)) = **O(2^n · n^3)**.
- n=16 기준 약 65536 × 16³ ≈ 2.7×10^8. 통과는 하지만 여유가 적다.

**✅ 잘 짚은 것**

- **복귀형 TSP의 차이 두 곳을 정확히 인지**했다: 시작 상태 세팅과 "복귀 간선을 더하지 않음".
- 주석에서 **"단방향(비대칭)이라 MST는 못 쓴다"** 를 스스로 배제한 판단이 정확하다. 무방향 그래프 가정이 깨지므로 MST/그리디 계열은 부적합하다.
- `dist[last][nx] < 0` 길 없음 처리, `INF = MAX/2`로 오버플로 방지, `dp[FULL-1][i]` 최솟값 집계 — 핵심 디테일을 모두 챙겼다.

**❌ 문제점 / 개선점**

- **외부 `for start` 루프가 불필요하다 (가장 큰 개선점).** `dp[mask][last]`는 정의상 "mask 안의 *어딘가*에서 출발해 last에 도달"을 이미 함의한다. 따라서 시작점마다 따로 돌릴 필요 없이, **모든 단일 지점 상태 `dp[1<<i][i]=0`을 한 번에 시드**하면 단 한 번의 DP로 같은 답이 나온다 → **O(2^n · n^3) → O(2^n · n^2)** 로 n배 단축.
- 부수적으로, 시작점마다 `dp` 배열 전체를 `Arrays.fill`로 재초기화하는 비용(n × 2^n)도 함께 사라진다.

---

### 모범 답안: Held-Karp (시작점 일괄 시드)

**알고리즘**

1. `dp[mask][last]`를 모두 `INF`로 초기화.
2. **출발 자유** → 모든 `i`에 대해 `dp[1<<i][i] = 0` 으로 시드 (외부 start 루프 제거).
3. `mask`를 1..FULL로 키우며, `mask`에 포함된 `last`에서 미방문 `nx`로 전이:
   `dp[mask|(1<<nx)][nx] = min(…, dp[mask][last] + dist[last][nx])` (길 없으면 skip).
4. **복귀 없음** → 전체 방문 상태 `dp[FULL][last]` 중 최솟값이 답. 전부 `INF`면 `-1`.

**day3(복귀형)과의 비교**

| | day3 복귀형 | day4 열린 경로형 |
|---|---|---|
| 시작 시드 | `dp[1][0]=0` (0번 고정) | `dp[1<<i][i]=0` 전부 |
| 정답 집계 | `dp[FULL][last] + dist[last][0]` 최소 | `dp[FULL][last]` 최소 (복귀 간선 없음) |

**시간복잡도**

| 단계 | 복잡도 |
|------|--------|
| dp 초기화 | O(2^n · n) |
| 상태 전이 (mask × last × nx) | O(2^n · n^2) |
| 정답 집계 | O(n) |
| 합계 | **O(2^n · n^2)** |

**공간복잡도**: O(2^n · n)

---

### 핵심 정리

- **Held-Karp DP에서 "출발점 자유"는 외부 루프가 아니라 시드(seed)로 표현한다.** `dp[1<<i][i]=0`을 전부 깔면 단일 DP로 모든 출발점이 동시에 처리된다 — 시작점마다 DP를 재실행하면 불필요하게 n배 느려진다.
- **복귀형 vs 열린 경로형의 차이는 딱 두 군데**: (1) 시작 시드, (2) 마지막 복귀 간선을 더하는가. 전이 루프 본체는 동일하다.
- **비대칭(단방향) 그래프에서는 MST/무방향 그리디가 성립하지 않는다.** TSP류는 `n ≤ ~20` 이면 비트마스크 DP가 정석.
- 비용 합산이 클 수 있으니 `INF`는 `Integer.MAX_VALUE / 2`로 두어 `dp + dist` 오버플로를 막는다.
