## 문제 1. 장을 다 보고 집으로

### 문제 요약

- N개의 장소, M개의 **양방향 가중치 도로**. 집은 1번. K개의 심부름.
- 심부름 i는 `S_i`에서 물건을 받아 `E_i`에 전달한다(받기/전달은 시간 0).
- 가방 용량 C: **동시에 들고 있는 물건 수 ≤ C** (전달하면 가방에서 사라짐).
- 집에서 출발 → 모든 심부름 완료 → 집으로 복귀하는 **최소 이동시간**. 불가능하면 -1.
- 제한: `2 ≤ N ≤ 100,000`, `1 ≤ M ≤ 200,000`, **`1 ≤ K ≤ 9`**, `1 ≤ C ≤ K`, `1 ≤ T ≤ 10^9`. 정답은 64비트 정수 범위.

---

### 내 풀이 분석

- **풀이 결과**: 실패 (미완성)

**알고리즘**

입력을 읽고 인접 리스트 `roads`까지만 구성한 단계에서 멈췄다.
main 주석에서 핵심 방향은 정확히 잡고 있었다:
- "완탐 절대 아님", "DP인 것 같다"
- "모든 지점 간 최단거리 필요 — 플로이드 워샬?"

**시간복잡도**

- 핵심 DP가 비어 있어 분석 불가.

**✅ 잘 짚은 것**

- 양방향 가중치 그래프를 인접 리스트로 올바르게 구성하려 한 점.
- "지점 간 최단거리 전처리가 먼저 필요하다"는 핵심 1단계를 정확히 인지한 점.
- 완전탐색이 안 된다는 것(상태 공간이 큼)을 스스로 판단한 점.

**❌ 문제점 / 개선점**

- **거리 전처리 선택 오류**: 플로이드-워샬은 `O(N^3)`이라 `N ≤ 100,000`에서 불가능.
  → 거리가 필요한 노드는 **집 + 모든 S_i, E_i (최대 2K+1=19곳)** 뿐이므로,
    그 노드들에서만 **다익스트라**를 돌려 거리를 압축해야 한다.
- **1-based 인덱스 OOB 버그**: `roads`를 `n` 크기로 만들었는데 `roads.get(a)`의 `a`는 1..n.
  → 크기를 `n+1`로 해야 한다.
- 가장 중요한 "어떤 순서로 받고 전달할지"를 결정하는 **상태 DP**가 통째로 비어 있음.
  → `K ≤ 9`라는 작은 제한이 **상태 압축 DP** 신호인데 활용하지 못함.

---

### 모범 답안: 거리 압축(다익스트라) + 3진 마스크 DP

**알고리즘**

1. **특수 노드 수집**: 집(1) + 모든 `S_i`, `E_i`. 중복 제거 후 최대 `2K+1 = 19`개. 각각 인덱스 부여.
2. **거리 압축**: 각 특수 노드에서 다익스트라 1회 → 특수 노드 사이의 최단거리 행렬 `dist[sp][sp]`.
   (전체 쌍이 아니라 "의미 있는 19개 노드" 사이 거리만 필요.)
3. **상태 정의**: 심부름마다 3가지 상태 → **3진수 마스크**.
   - 0 = 미수령, 1 = 수령했고 미전달(가방 점유), 2 = 전달완료. 전체 상태 수 `3^K ≤ 19683`.
   - `dp[mask][pos]` = 상태가 `mask`이고 현재 특수 노드 `pos`에 있을 때 최소 이동시간.
4. **전이** (`dp[0][집] = 0`에서 시작):
   - 가방 보유 수 `carrying` = 상태가 1인 심부름 개수.
   - `carrying < C`이면 상태 0인 심부름 i를 **수령**: `S_i`로 이동(`dist[pos][S_i]`), 자리 i를 0→1.
   - 상태 1인 심부름 i를 **전달**: `E_i`로 이동(`dist[pos][E_i]`), 자리 i를 1→2.
   - 두 전이 모두 마스크 값이 `+3^i` 만큼 **증가** → `mask` 오름차순 1회 순회로 위상 정렬 자동 충족.
5. **정답**: 모든 자리가 2인 `full` 상태에서 각 `pos`에 대해 `dp[full][pos] + dist[pos][집]`의 최솟값.
   도달 불가(INF)면 -1.

**예시 추적** (`6 7 2 2`, 심부름 (2,6),(3,5))

```
C=2라 두 물건을 동시에 들 수 있다.
1 →(2) 2 : e0 수령        누적 2
2 →(2) 3 : e1 수령        누적 4   (가방 2개, C=2 OK)
3 →(4) 5 : e1 전달        누적 8   (3-4-5)
5 →(2) 6 : e0 전달        누적 10  (5-6)
6 →(7) 1 : 귀가           누적 17  (6-5-2-1 = 2+3+2)
정답 = 17
```

**시간복잡도**

| 단계 | 복잡도 |
|------|--------|
| 특수 노드 다익스트라 (≤2K+1회) | O((2K+1) · M log N) |
| 거리 행렬 채우기 | O((2K+1)^2) |
| DP 전이 | O(3^K · (2K+1) · K) |
| 합계 | O(K · M log N + 3^K · K^2) |

`3^9 · 19 · 9 ≈ 3.4×10^6`, 다익스트라 19회로 충분히 빠르다.

**공간복잡도**: O(N + M + 3^K · (2K+1))

---

### 핵심 정리

- **`K ≤ 9`(혹은 ≤ 20)처럼 유난히 작은 제한 = 상태 압축 DP 신호.** 각 원소가 2가지면 2진(비트마스크),
  이번처럼 3가지(미수령/들고있음/완료)면 **3진 마스크**를 쓴다.
- **거리 전처리는 "필요한 노드 수"로 방법을 고른다.** 모든 쌍이 필요하고 N이 작으면(≤ ~400) 플로이드,
  관심 노드가 소수면 **그 노드들에서만 다익스트라**로 거리를 압축한다. (이번 문제의 핵심 함정)
- 용량/제약(가방 C)은 **상태에서 파생되는 값**(상태 1의 개수)으로 자연스럽게 표현하면 별도 차원이 필요 없다.
- 전이가 항상 마스크 값을 증가시키면 `mask` 오름차순 순회만으로 위상 정렬이 보장된다.
- 거리·누적시간은 `T ≤ 10^9`, 경로가 길어질 수 있으므로 **`long` + 안전한 INF(`MAX/4`)** 로 오버플로 방지.
- 1-based 노드는 배열/리스트를 **`n+1`** 크기로 — 사용자가 낸 OOB 버그 단골 패턴.

## 문제 2. 임계 네트워크

### 문제 요약

- 무방향 그래프. 각 간선은 위험도 `w`와 통행 비용 `c`를 가진다.
- 정수 `X`를 정하면 **위험도 `w ≤ X`인 간선만** 사용 가능.
- 그 간선들만으로 1→N 경로 중 **비용 합 ≤ K**인 경로가 존재하도록 하는 **최소 X**를 구한다. 불가능하면 -1.
- 제한: `2 ≤ N ≤ 100,000`, `1 ≤ M ≤ 300,000`, `1 ≤ K ≤ 10^18`, `1 ≤ w, c ≤ 10^9`, 중복 간선 가능.

---

### 내 풀이 분석

- **풀이 결과**: 부분 성공 (방향은 정확, 핵심 구현 미완성 + 설계 버그)

**알고리즘**

main 주석에서 핵심 전략을 정확히 적어두었다:
- "1→N 최소거리(다익스트라)", "X 구하기: 이진탐색".

즉 **이분 탐색 + 다익스트라**라는 정답 골격을 스스로 도출했다. 다만 `dijkstra`와
이분 탐색 루프 본문이 비어 있고, 거리 자료구조 설계에 치명적 문제가 있다.

**시간복잡도**

- 의도한 설계(이분 탐색 × 다익스트라)는 `O(log D · M log N)`로 적절. 다만 구현 미완성.

**✅ 잘 짚은 것**

- "**위험도 X에 대한 단조성 → 이분 탐색**"이라는 핵심 관찰을 정확히 함.
- 각 X에서의 판정을 **다익스트라(최소 비용)** 로 연결한 것.
- 입력 그래프를 `n+1` 크기로 올바르게 구성하고 양방향 간선을 모두 넣음 (Solution1의 OOB 실수 교정).
- 간선 정보를 `{to, w, cost}` 3-튜플로 들고 간 것.

**❌ 문제점 / 개선점**

- **치명적 메모리 버그**: `long[][] dist = new long[n+1][n+1]`은 `O(N^2)`.
  `N=100,000`이면 `10^10` 칸 → 메모리 초과. 다익스트라 거리는 **1차원 `dist[n+1]`** 이면 충분.
- **이분 탐색 대상 오류**: `r = 1e9 * (n-1)`은 **비용**의 상한인데, 탐색해야 할 값은 **위험도 X**다.
  → 탐색 범위는 위험도 `w`(1..1e9), 혹은 **서로 다른 w 값들을 정렬한 배열** 위에서 진행해야 한다.
- **부등호 오류**: `road[1] < x`는 `w < X`. 문제 조건은 `w ≤ X` → `<=` 여야 한다.
- 이분 탐색 루프 본문(`mid`로 판정 후 `l`/`r` 갱신, answer 기록)과 `dijkstra` 본문이 비어 미완성.
- 우선순위 큐 비교자가 `comparingInt`인데, 누적 비용은 `long`(최대 `1e9 × 1e5 = 1e14`)이라 `int` 비교는 오버플로 위험.

---

### 모범 답안: 위험도 이분 탐색 + 비용 다익스트라

**알고리즘**

1. 간선의 **서로 다른 위험도 값**을 모아 오름차순 정렬 → 후보 `cand[]`.
   (X를 키우면 쓸 수 있는 간선만 늘어나므로 feasible 여부가 단조 → 이분 탐색 가능.)
2. `cand` 인덱스에 이분 탐색: 중앙값 `X = cand[mid]`에 대해 `feasible(X)` 판정.
3. `feasible(X)`: `w ≤ X`인 간선만 사용해 1→N **최소 비용 다익스트라**, 결과 비용 `≤ K`면 true.
4. true면 정답 후보로 기록 후 `hi = mid-1`(더 작은 X 시도), false면 `lo = mid+1`.
5. 끝까지 못 찾으면 -1.

**예시 추적** (`5 6 10`, 정답 7)

```
서로 다른 w = {1,2,3,4,5,7}
X=3: 쓸 수 있는 간선으로 1→3→5 비용 8+5=13 > 10  → false
X=4: 1→2(c3)만 추가, 2→5(w7)는 불가 → 1→N 경로 비용 여전히 >10 → false
X=7: 1→2(c3)→5(c4)=7 ≤ 10 → true
최소 X = 7
```

**시간복잡도**

| 단계 | 복잡도 |
|------|--------|
| 위험도 정렬/중복 제거 | O(M log M) |
| 이분 탐색 × 다익스트라 | O(log D · M log N) |
| 합계 | O(M log M + log D · M log N) |

`D` = 서로 다른 위험도 개수(≤ M). `3×10^5 · 18 · 18 ≈ 10^8` 수준.

**공간복잡도**: O(N + M)

---

### 핵심 정리

- "**임계값 X를 정하고, 그 조건에서 어떤 성질이 성립하는 최소/최대 X**" → 단조성 확인 후 **파라메트릭 서치(이분 탐색)**.
  바깥은 이분 탐색, 안쪽은 그 X에서의 판정(여기선 다익스트라)으로 분리하는 패턴이 정석.
- **이분 탐색의 대상 변수를 명확히 하자.** 이번엔 답이 "위험도 X"인데 "비용" 범위로 탐색하면 틀린다.
  답이 될 수 있는 값(간선 위험도)들만 모아 그 위에서 탐색하면 깔끔하다.
- 다익스트라 거리 배열은 **1차원 `dist[N]`**. `dist[N][N]` 같은 2차원은 N이 크면 즉사(메모리 초과).
- 누적 비용·K는 `10^14`~`10^18`이므로 **전부 `long`**, PQ 비교자도 `Long.compare` 사용 (int 오버플로 주의).
- 부등호(`≤` vs `<`)는 문제 문장 그대로 — "w ≤ X"를 `<`로 적는 실수 단골.
