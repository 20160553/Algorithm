## 문제 1. 연속 선택 제한 최대 점수

### 문제 요약

- 길이 N인 수열에서 왼쪽→오른쪽으로 각 위치를 선택/건너뛴다.
- **연속으로 3개 이상은 선택할 수 없다** (최대 2개 연속까지 허용).
- 선택한 위치들의 점수 합의 최댓값을 구한다.
- 제한: 1 ≤ N ≤ 100,000, 1 ≤ Ai ≤ 10,000

---

### 내 풀이 분석

- **풀이 결과**: 부분 성공 — 코드는 완성됐으나 예제에서 오답(33이 정답인데 **37 출력**)

**알고리즘**

직전 3칸의 선택 여부를 3비트 마스크(0~6, 111은 연속3 금지로 제외)로 들고 가는 비트마스크 DP를 시도했다. `dp[i][mask]`에서 새 위치 i의 선택 여부 `j`를 윈도우 맨 위 비트로 밀어 넣고(`nm = j<<2 | mask>>1`), 이전 윈도우 상위 2비트가 모두 1이면 연속3이 되므로 건너뛰는 구조다. 상태 설계 방향(슬라이딩 윈도우 + 연속2 제한)은 정확하다.

**시간복잡도**

- O(N × 2 × 7) = O(N). 복잡도 자체는 통과 가능한 수준.

**✅ 잘 짚은 것**

- "3개 중 1개는 무조건 버린다"는 직관과 윈도우 마스크 전이라는 큰 그림은 맞다.
- 연속3 금지를 마스크 상위 비트 검사로 잡으려는 접근은 옳은 방향.

**❌ 문제점 / 개선점**

- **미선택(j=0)에도 점수를 더함 (핵심 버그)**: `j` 루프 안에서 `j` 값과 무관하게 항상 `score + prev`를 더한다. j=0은 위치 i를 **선택하지 않는** 경우이므로 `arr[i]`를 더하면 안 된다. → 과대계산으로 37 출력.
- **skip 조건이 미선택 경로까지 폐기**: `if ((mask>>1)==3) continue;` 가 j와 무관하게 동작한다. 직전 두 칸이 11이어도 이번 칸을 **고르지 않으면**(j=0) 윈도우는 (i-2,i-1,i)=(1,1,0)으로 합법인데, 이 전이를 통째로 버린다. skip은 j==1일 때만 적용돼야 한다.
- **경계 인덱스 예외**: `dp[2][...]`부터 초기화하므로 N이 1 또는 2면 `ArrayIndexOutOfBounds`가 난다.
- **상태 과설계**: 실제로 필요한 정보는 "i에서 끝나는 연속 선택 길이(0/1/2)" 3가지뿐인데 7상태로 키워 버그 표면적을 늘렸다.

---

### 모범 답안: 연속 길이 상태 DP

**알고리즘**

상태를 "현재 위치에서 끝나는 연속 선택 개수 k(0/1/2)"로만 정의한다.

1. `dp[i][0]` = i 미선택 → 직전 상태 무엇이든 가능 = `max(dp[i-1][0], dp[i-1][1], dp[i-1][2])`
2. `dp[i][1]` = i 선택 + 직전 미선택 = `dp[i-1][0] + A[i]`
3. `dp[i][2]` = i 선택 + 직전도 선택(연속2) = `dp[i-1][1] + A[i]`  (연속3 금지이므로 `dp[i-1][2]`는 사용 불가)
4. 답 = `max(dp[N-1][0..2])`. 직전 행만 필요하므로 변수 3개(`p0,p1,p2`)로 롤링한다.

**예시 추적** (A = 6 10 13 9 8 1)

```
i=0(6) :  p0=0   p1=6    p2=-inf
i=1(10):  p0=6   p1=10   p2=16
i=2(13):  p0=16  p1=19   p2=23   ← 13 선택 시 연속3(6,10,13) 차단 후 최적
i=3(9) :  p0=23  p1=25   p2=28
i=4(8) :  p0=28  p1=31   p2=33   ← 9,8 연속2 선택
i=5(1) :  p0=33  p1=29   p2=32
답 = max(33,29,32) = 33
```

**시간복잡도**

| 단계 | 복잡도 |
|------|--------|
| 전체 위치 1회 순회 | O(N) |
| 위치당 상태 전이 | O(1) |
| 합계 | O(N) |

**공간복잡도**: O(1) (롤링 변수 3개)

---

### 핵심 정리

- "연속 K개까지만 허용" 류 DP는 **마스크가 아니라 '끝에서의 연속 길이'를 상태로** 잡으면 상태 수가 K+1개로 줄어 가장 단순하다.
- DP 전이에서 "선택 안 함" 분기는 **점수를 더하지 않는다** — 선택/미선택 분기에서 가중치 적용 여부를 꼭 분리할 것.
- 누적/슬라이딩 DP는 `dp[2]`처럼 특정 인덱스부터 채우면 작은 N에서 인덱스 예외가 나기 쉽다. base를 변수 초기값으로 두고 i=0부터 일반 루프로 도는 편이 안전하다.

## 문제 2. 컨베이어 배송 계획

### 문제 요약

- N개의 상자를 **순서 변경 없이 앞에서부터** 트럭에 싣는다. 용량 X를 넘지 않는 동안 계속 싣고, 넘는 순간 출발(하루 1회).
- 모든 상자를 **D일 이내**에 운반할 수 있는 **최소 용량 X**를 구한다.
- 제한: 1 ≤ D ≤ N ≤ 100,000, 1 ≤ Wi ≤ 10,000 (총합 최대 10^9)

---

### 내 풀이 분석

- **풀이 결과**: 성공 — 예제 1(15), 예제 2(3) 및 경계(N=1, 1일에 전부) 모두 정답

**알고리즘**

답 X에 대한 **매개변수 탐색**. `l=1, r=sum[n]` 구간을 이분하며, 각 `mid`에 대해 `solve(mid)`로 D일 이내 가능 여부를 판정한다. `solve`는 prefix sum과 투포인터(`l`, `r`)로 현재 적재 무게 `sum[r]-sum[l]`가 X를 넘으면 그 날을 마감(`day++`, `l=r-1`)하는 그리디다. 가능하면 `r=mid-1`로 더 줄이고, 불가능하면 `l=mid+1`.

**시간복잡도**

- 판정 O(N), 이분탐색 O(log(총합)) → 전체 **O(N log ΣW)**. 제한 내 충분히 통과.

**✅ 잘 짚은 것**

- "X가 클수록 필요 날수는 단조 감소"라는 단조성을 정확히 포착해 매개변수 탐색으로 환원했다.
- 단일 상자가 X보다 무거운 경우(`l=r-1`로 r 정지) `day`가 계속 증가해 자연히 `false`로 종료 — 무한 루프를 피한 점이 좋다.
- prefix sum으로 구간합을 O(1)에 처리.

**❌ 문제점 / 개선점**

- **`mid = (l+r)/2` 오버플로 경계**: `l+r`이 최대 약 2×10^9까지 커져 int 상한(약 2.147×10^9)에 아슬아슬하다. 지금 제한에선 통과하나, `mid = l + (r-l)/2`로 쓰는 습관이 안전하다.
- **탐색 하한이 느슨**: `l=1`로 시작하지만 답은 절대 `max(Wi)` 미만일 수 없다. 하한을 `maxW`로 잡으면 탐색 구간이 줄고 의미도 명확해진다.
- **투포인터 그리디가 과설계**: prefix sum + `l/r` 포인터 + 루프 밖 마지막 `++day` 보정은 추론이 까다롭다. 단순 누적(`load`) 그리디로도 동일 O(N)이며 경계 실수 여지가 적다.

---

### 모범 답안: 매개변수 탐색(parametric binary search) + 누적 그리디

**알고리즘**

1. 하한 `lo = max(Wi)`, 상한 `hi = ΣWi`로 설정한다.
2. `lo < hi` 동안 `mid = lo + (hi-lo)/2`를 잡고 `feasible(mid)` 판정.
3. `feasible(cap)`: 앞에서부터 누적 `load`에 더해 `cap` 초과 시 `days++`, `load=0` 후 다시 적재. `days > d`면 `false`.
4. 가능하면 `hi = mid`(더 줄이기), 불가능하면 `lo = mid+1`. 수렴한 `lo`가 답.

**시간복잡도**

| 단계 | 복잡도 |
|------|--------|
| feasible 1회 | O(N) |
| 이분탐색 횟수 | O(log ΣW) |
| 합계 | O(N log ΣW) |

**공간복잡도**: O(N) (무게 배열)

---

### 핵심 정리

- "최댓값을 최소화 / 최솟값을 최대화 + 단조 판정"이 보이면 **매개변수 탐색**이 정석. 판정 함수의 단조성만 확인하면 된다.
- 이 유형의 탐색 하한은 보통 `max(원소)`, 상한은 `sum(원소)` — 답의 물리적 의미에서 경계를 잡으면 구간이 깔끔해진다.
- 이분탐색 중간값은 습관적으로 `lo + (hi-lo)/2`로 써서 합 오버플로를 원천 차단한다.
