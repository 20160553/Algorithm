# 알고리즘 풀이 리뷰 & 모범답안 생성

인수: $ARGUMENTS  
형식: `{dayFolder} {SolutionFile}` (예: `day2 Solution1`)

## 작업 순서

### 1. 대상 파일 파악

인수를 파싱해 dayFolder와 SolutionFile을 추출한다.  
`src/com/algo/{dayFolder}/{SolutionFile}.java` 파일을 읽는다.

### 2. 파일 내용 분석

파일에서 다음을 파악한다:
- **문제 설명**: 주석 블록의 문제 내용 (입력/출력/제한)
- **사용자 풀이 코드**: 사용자가 작성한 Solution 클래스
- **이미 모범답안이 있는지**: `{SolutionFile}Answer` 클래스가 이미 존재하면 모범답안 생성을 건너뛴다
- **사용자의 접근 방식**: main 내 주석 및 코드 구조 파악

### 3. 모범답안 생성 및 추가

`{SolutionFile}Answer` 클래스가 없는 경우에만 생성한다.

**모범답안 코드 작성 기준:**
- 올바른 알고리즘으로 문제를 완전히 풀어낸다
- 파일 상단에 이미 `import java.util.*; import java.io.*;`가 있으면 중복 추가하지 않는다
- 클래스 위 주석 블록에 다음을 포함한다:
  - 핵심 알고리즘 아이디어 (1~3줄)
  - 사용자 풀이 대비 개선점 (있을 경우)
  - 시간복잡도 / 공간복잡도
- 코드 내 인라인 주석은 WHY가 비자명한 경우에만 한 줄로 작성한다
- 코드는 `{SolutionFile}.java` 파일 **끝**에 이어서 추가한다

### 4. 날짜 파일 찾기 또는 생성

`src/com/algo/{dayFolder}/` 폴더 안에서 `YYYY.MM.DD.md` 패턴의 파일을 검색한다.

- **파일이 있으면**: 해당 파일에 내용을 이어서 작성한다
- **파일이 없으면**: dayFolder의 생성 날짜를 기준으로 `YYYY.MM.DD.md` 파일명을 결정한다  
  (생성 날짜를 알 수 없으면 오늘 날짜를 사용한다)

확장자 없는 날짜 파일(예: `2026.06.14`)이 폴더에 있으면 내용을 참고만 하고, 새 `.md` 파일에 작성한다.

### 5. 날짜 파일에 분석 내용 작성

기존 `.md` 파일에 내용이 있으면 구분선(`---`) 뒤에 이어서 추가한다.  
없으면 새 파일로 시작한다.

아래 Markdown 형식을 따른다 (한국어로 작성):

````markdown
## 문제 N. {문제 제목}

### 문제 요약

- (핵심 조건 3~5줄)
- 제한: N, M 등

> ⚠️ 예시 오류: (예시 출력이 잘못된 경우에만 작성, 없으면 이 항목 생략)

---

### 내 풀이 분석

- **풀이 시간**: (코드 주석에서 파악, 없으면 생략)
- **풀이 결과**: 성공 / 실패 / 부분 성공 — 코드 완성도 기준

**알고리즘**

(사용자가 시도한 핵심 알고리즘 설명)

**시간복잡도**

- (분석)

**✅ 잘 짚은 것** (있으면 작성, 없으면 항목 생략)

- ...

**❌ 문제점 / 개선점**

- (버그, TLE 위험, 설계 오류, 미완성 등 항목별로 정리)

---

### 모범 답안: {알고리즘 이름}

**알고리즘**

1. Step 1
2. Step 2

**예시 추적** (예시가 직관적으로 이해하기 어려운 경우에만 포함)

```
...
```

**시간복잡도**

| 단계 | 복잡도 |
|------|--------|
| ... | O(...) |
| 합계 | O(...) |

**공간복잡도**: O(...)

---

### 핵심 정리

- (이 문제 유형의 패턴 정리)
- (다음에 비슷한 문제가 나왔을 때 기억해야 할 것)
````

## 주의사항

- 날짜 파일 내 동일 문제(`## 문제 N.`)가 이미 있으면 덮어쓰지 말고 "이미 작성된 내용이 있습니다"라고 알린다
- 문제 번호 N은 파일명에서 추출한다 (Solution1 → 문제 1)
- 문제 제목은 문제 주석의 `문제:` 행에서 가져온다
- 풀이 결과 판단 기준: Solution 클래스가 비어 있거나 미완성이면 "실패", 완성됐지만 TLE/오류 있으면 "부분 성공", 정상 완성이면 "성공"
